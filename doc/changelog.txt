Z_COM 更新日志
===============

V0.0.8 (2026-05-22)
-------------------
新功能：
- 更多设置重构，统一拼包超时配置入口
- 完善 Nuitka 打包脚本，优化构建流程和体积
- 实例号分配改用文件锁，移除 psutil 依赖
- 关于对话框显示编译时间

改进：
- 界面微调：统一中英文菜单文字，优化控件间距
- RTT/Socket 读取线程优化，提升稳定性
- 扩展发送面板交互优化
- 日志模块优化，路径处理更加健壮
- UI 布局微调，串口设置对话框重构

V0.0.7 (2026-05-21)
--------------------
新功能：
- 新增 Socket 通信支持（TCP/UDP，客户端/服务器模式）
- 新增 ANSI 颜色显示解析，支持终端控制字符渲染
- 新增自动化测试框架，包含单元测试和端到端测试

改进：
- 重构项目架构，拆分为 core/io/windows 三层模块
- 优化 ASCII 显示模式，自动断行显示多行数据
- UI 布局优化，状态栏集成端口选择和配置

V0.0.6 (2026-05-21)
--------------------
改进：
- 新增 Nuitka 打包脚本（pack_nuitka.py）
- 优化打包流程

V0.0.5 (2026-05-20)
--------------------
改进：
- 重构扩展发送模块，统一RTT和串口发送接口
- 新增启动发送按钮，支持四种发送模式：
  - 单条发送：发送序号最小的条目一次
  - 循环单条：循环发送序号最小的条目
  - 多条发送：按序号依次发送，完成后停止
  - 循环多条：按序号依次发送，循环不停
- 发送失败时自动停止并恢复按钮状态
- 序号为0的条目不参与发送
- 移除enabled字段，简化数据结构

V0.0.4 (2026-05-20)
--------------------
新功能：
- 支持程序多开，自动识别实例号（使用 psutil 检测同名进程数量）
- 实例1 使用根目录配置和日志，实例2+ 自动创建 instance_X 目录
- 窗口标题显示实例号便于区分（如：Z_COM V0.0.4 [实例2]）

改进：
- ASCII 显示模式下，不可见字符显示为特殊符号（如 ␍␊ 等）
- 换行符（LF）在显示时会实际换行，便于阅读
- 默认 J-Link 连接时不勾选复位选项

V0.0.3 (2026-05-20)
--------------------
新功能：
- 集成 J-Link RTT 功能，支持通过 J-Link 进行 RTT 数据收发
- 串口选择下拉框同时显示普通串口和 J-Link 设备（格式：JLINK:SN=xxx）
- 更多设置中新增 RTT 设备配置区域（芯片型号、J-Link 速度、连接时复位、RTT 地址范围）
- 芯片型号支持手动输入，连接成功后自动保存到历史记录
- J-Link 设备扫描在后台线程执行，避免阻塞界面

依赖：
- 新增 pylink-square 依赖（用于 J-Link RTT 通信）
- 新增 psutil 依赖（用于进程检测，支持多开）

V0.0.2 (2026-05-19)
--------------------
新功能：
- 扩展发送功能，支持多条数据批量发送
- 支持单条发送、多条顺序发送、循环发送模式
- 每条数据可设置独立延时（0-60000ms）
- 扩展发送配置支持导入导出（JSON格式）

改进：
- 优化串口保护锁，串口打开时禁用刷新按钮和串口选择
- 支持配置热更新，修改波特率等参数后立即生效
- 优化数据帧自动拼接（50ms超时）

V0.0.1 (2026-05-18)
--------------------
初始版本：
- 基于 PySide6 开发的串口调试工具
- 支持 HEX/ASCII/HEX+ASCII 多种显示模式
- 支持数据帧自动拼接
- 支持自动发送和回车换行
- 自动记录日志（按日期创建）
- 参数自动保存和加载
