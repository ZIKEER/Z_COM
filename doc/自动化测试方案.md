## 6. Socket Reader 架构说明

### 6.1 Windows `select()` 兼容性问题

`SocketReaderThread` 最初使用 `select.select()` 实现 I/O 多路复用，但在 **Windows + QThread** 环境下发现以下问题：

| 问题 | 表现 | 根因 |
|---|---|---|
| `select()` 不返回 | `QThread.wait()` 永久挂起 | Windows 上从另一个线程关闭 socket 时 `select()` 不会立即返回 |
| 信号断开阻塞 | `disconnect()` 死锁 | Qt 跨线程信号断开可能等待 sender 线程，但 sender 卡在 `select()` |
| QThread 析构阻塞 | 设置 `reader_thread = None` 时 Python GC 销毁 QThread，析构 `wait()` 死锁 | QThread 析构函数等待线程结束，但线程不结束 |
| `bool running` 不可见 | 线程永远看不到 `running = False` | 无内存同步保障，寄存器/CPU 缓存可能导致 stale 值 |

### 6.2 修复方案（v2 — 当前实现）

**放弃 `select.select()`，改用非阻塞 socket + 轮询模式**：

```
TCP Client:  recv(65536) → BlockingIOError → msleep(20) → recv() 循环
TCP Server:  accept() → BlockingIOError → 遍历 client recv() → msleep(20) 循环
UDP:         recvfrom(65536) → BlockingIOError → msleep(20) → recvfrom() 循环
```

- 用 `threading.Event`（`_stop_event`）替代 `bool running`，保证线程间有序性
- `stop()` 先 `_stop_event.set()`，再 `self._sock.close()`
- `_disconnect_internal()` 先断开所有 Qt 信号连接，再 `stop()`，再 `wait(2000)`

### 6.3 性能影响

| 指标 | select() 方案（已放弃） | 轮询方案（当前） |
|---|---|---|
| 响应延迟 | 事件驱动，~1ms | 固定 20ms 轮询间隔 |
| 线程退出延迟 | `stop()` 到线程退出最坏 100ms | `stop()` 到线程退出最坏 30ms |
| Windows 兼容 | 不兼容（QThread 下死锁） | 完全兼容 |
| CPU 使用 | select 空闲时 0% | 20ms 轮询约 5% |

对串口/RTT/Socket 调试工具来说，20ms 轮询延迟完全可以接受。

## 7. 已知问题

### 7.1 `test_main_window.py` 全部 ERROR

所有 18 个 `test_main_window.py` 用例因同一原因失败：

```
NameError: name 'APP_NAME' is not defined
src/windows/main_window.py:50
```

**修复方法**：在 `main_window.py` 顶部补全 import：

```python
from src.version import APP_NAME, VERSION
```

### 7.2 `select()` 已从代码中移除

`socket_reader.py` 不再引用 `select` 模块，`import select` 已删除。

### 7.3 QThread 析构时可能触发错误

Python GC 回收 `SocketReaderThread` 对象时，如果线程仍在 `msleep(20)` 中未退出，可能触发：

```
Error calling Python override of QThread::run()
```

当前方案：`_disconnect_internal()` 先用 `wait(2000)` 等待线程退出，再设置 `reader_thread = None`。

## 8. CI 集成建议

```yaml
name: Test
on: [push, pull_request]
jobs:
  test:
    runs-on: windows-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-python@v5
        with:
          python-version: "3.11"
      - run: pip install PySide6 pyserial pylink-square pytest pytest-qt
      - run: python -m pytest tests/ --ignore=tests/test_main_window.py -v
```

> 说明：CI 跳过 `test_main_window.py`（预存 bug）+ 硬件测试（无物理串口/JLink）。纯逻辑 + mock 传输层 + Socket 回环 E2E 够覆盖核心风险。
